      Updates & Roadmap

      Latest shipped changes plus known gaps and planned follow-up grouped by workspace page.

      What's New

      Shipped updates that are available now (latest releases).

      Version 0.9.0

      • Composer Advanced Controls now support opt-in custom scope resolution, with per-resource scope toggles and precedence controls for champion metadata, champion tags, tag definitions, requirements, and compositions.
      • Composer generation and requirement evaluation now run against the selected scoped champion truth instead of assuming the global catalog only.
      • OR-linked requirement scoring now uses joiner-aware aggregation so inactive alternate branches no longer inflate missing coverage or branch scores in Review and tree ranking.
      • Draft saves now use dedicated Save Draft and Load Draft modals, with optional descriptions and full Composer-state reload support.
      • Tags workspace now opens tag-definition promotion requests from the non-global tag editor, shows the request note in a modal, and lets requesters cancel pending submissions while review queues stay visible in-page.
      • My Champions now shows collapsed one-click add notifications when Riot mastery/history suggests a strong recent champion that fits the selected role but is not yet in that role’s pool.

      Version 0.8.8

      • Report an issue now opens a blocking in-app modal from any page, captures the current page/tab context, and can submit directly to GitHub when server-side issue reporting is configured; otherwise it falls back to the external GitHub issues page.
      • Users workspace now supports search, role filtering, and bulk member/global permission updates from the existing admin directory.
      • Teams discover, join-request, and invitation sections now auto-refresh when Teams opens, use compact last-refreshed telemetry with inline refresh icons, and ship a repair migration plus startup guard for drifted invitation schema so broken databases are corrected or blocked before the Teams page loads.
      • Tags workspace now supports `User`, `Team`, and `Global` scope selection, with global fallback so inherited definitions can be reviewed and overridden in-place.
      • Saving an inherited tag from `User` or `Team` scope now creates a scoped tag definition instead of mutating the global catalog, matching the existing scoped champion-profile workflow.
      • Champion tag editing now loads the tag catalog for the selected profile scope, so user/team-scoped tag definitions are available when editing those scoped champion tags.

      Version 0.8.7

      • Draft Path replaces Tree Map: the SVG dot graph is replaced by a 5-column pick-order layout where each column lists champion options for that role, styled like pool snapshot cards.
      • Hover any champion in the Draft Path to see a compact tooltip with viable finishes, coverage status, impact summary, and missing coverage.
      • Draft Path shows ALL champions per role across the entire tree; clicking a champion filters other columns to compatible picks without locking other roles. Selections sync bidirectionally with pool snapshot cards.
      • Composer actions now live in a sticky footer: Start Draft, Save Draft, Load Draft, and Clear Draft stay visible at the bottom while tree controls, Draft Path, and summary remain hidden until Start Draft is clicked.
      • Generation Stats and Draft Picks now open in dark-themed tech modals via eye-icon reveal rows instead of inline details/card lists.
      • Density dropdown and "Summary mode" notice removed; summary is now the default view.
      • Tree map legend hidden; panel description simplified to "Hover for details."
      • The Composer sticky action bar uses a gradient backdrop and respects the left nav width.
      • Branch card and tree node "Inspect" buttons renamed to "Select."

      Version 0.8.6 (Shipped)

      • Comfort Level dropdown restyled to match role-pill color scheme and compact View pill sizing.
      • Comfort Level filter dropdown min-width set to prevent shrinking when a short value is selected.
      • Composer stage-guide header panel removed; primary Composer actions now live in the sticky bottom action bar.
      • Power Spikes interaction reworked: clicking inside an existing range now modifies that range instead of creating a new one on top; clicking a single-level range removes it; dragging from a selected level redraws that range in place.
      • Profile Riot Champion Stats redesigned: top 3 most-played champions displayed as portrait cards in a 3-column grid inside a single container; hint summary text removed for the normal state.
      • Power Spike ranges that overlap or touch now auto-merge into a single range (e.g. dragging range 2 to be adjacent to range 1 combines them).

      Version 0.8.5 (Shipped)

      • Power Spikes replace Early/Mid/Late effectiveness: champion profiles now use drag-to-select level 1–18 ranges (up to 2 ranges per role), with a full-width ruler-style progress bar on cards and a multi-select level filter on the Champions grid.
      • Power spike ruler on champion cards now spans the full card width with visible tick marks and green progress bars for selected ranges.
      • My Champions comfort dropdown z-index fixed so it no longer appears behind cards.
      • Grid card scope and editor scope are now wired together: clicking Edit on a Global card opens the editor in Global scope; changing scope in the editor (e.g. to Team) updates the card's view scope on save or cancel.
      • All existing power spike / effectiveness data reset to empty so profiles can be re-entered from scratch with the new UI.
      • Scope panel redesigned as compact inline row: "Editing [dropdown] Scope [info icon]" with narrower dropdown, orange circle-i tooltip, and bottom border aligned with the right meta section.
      • Signed-in-as status line removed from page headers.
      • My Champions comfort filter now supports multi-select (e.g. S + A together) with checkbox-style dropdown and clear.
      • My Champions redesigned: selected champions now display as visual cards with champion portrait, Comfort Level pill selector (S/A/B/C), Riot mastery stats, and collapsible tags.
      • Champion pool selector replaced with a visual two-column modal (Available / Selected) featuring champion portrait icons, search filtering, and click-to-toggle selection.
      • My Champions cards now include name and comfort level filters to quickly find champions in your pool.
      • Riot mastery stats now load for all champions (not just top 5), so every champion in your pool can display mastery level and points.
      • Comfort Level dropdown now shows grade letter only (S/A/B/C) with full description on hover tooltip; non-selected options display with a white background.
      • Scope selector on Edit Champions cards now uses a compact "View" pill dropdown in the meta section instead of stacked indicator buttons.
      • Requirements and compositions now support explicit `User`, `Team`, and `Global` scope selection, using the same scope model as champion profile editing.
      • The Requirements and Compositions workspaces now load, create, update, and delete entries within the selected scope instead of forcing all catalog work through the global library.
      • Added a database-backed `champion_core` baseline imported from the provided processor dataset, preserving stable Riot/Data Dragon champion identity and numeric stat fields outside the draft-tag model.
      • Added an admin-only Champion Core inspection workspace so the imported DB rows can be verified directly in the deployed app before draft/API integration work begins.
      • Profile avatar selection now persists for signed-in users through the API and restores from backend profile data instead of relying on browser-local storage alone.
      • Profile now has a saved Display Team selector, so the team shown in the profile header can be chosen independently and restored from the API.

      Version 0.8.4 (Shipped)

      • Profile Riot Champion Stats now surface a featured most-played champion card instead of a placeholder summary when Riot data is available.
      • Riot mastery entries are now enriched with local champion names so the profile card and follow-up list can render readable champion labels.
      • Profile Riot Champion Stats now distinguish unavailable, unlinked, not-found, and API-error states with user-facing copy instead of a generic "not implemented" message.

      Version 0.8.3 (Shipped)

      • Champion profile editing now supports explicit `User`, `Team`, and `Global` scope selection from the editor header.
      • Champion cards now use always-visible visual scope indicators, and clicking an available `User` or `Team` indicator switches the card preview to that scoped metadata.
      • Scoped champion metadata now loads and saves per selected scope instead of forcing all metadata edits through the global profile only.
      • Edit Champions filters now include custom-metadata presence, so you can narrow the grid to champions with user or team metadata overrides.

      Version 0.8.2 (Shipped)

      • Other Roles modal pills now use the same compact role-pill style as the Edit Champions page; modal width reduced for a tighter layout.
      • Profile avatar in the left nav now fills the full width of the nav column, matching the logout link width.

      Version 0.8.1 (Shipped)

      • Avatar modal now displays champion portrait images with names for visual identification; images load from Riot Data Dragon.
      • Avatar selection now persists across page refreshes via local storage.
      • Saving the avatar via the modal now updates the profile avatar circle immediately with the selected champion portrait.
      • Account settings (First Name, Last Name) now persist to the server when saved; values survive page refresh.
      • Account editor buttons (Change Password, Save Account) now have top spacing and are right-aligned.

      Version 0.8.0 (Shipped)

      • Profile page redesigned as an identity showcase with pride header (avatar, summoner name, roles, team display) and a clickable settings list (accordion-style editors).
      • Champion avatar selection: click the avatar circle to open a modal picker with search filtering; select a champion then Save to set your portrait.
      • Team identity display: team name and large logo shown on the right side of the profile header; clicking either navigates to the Teams tab.
      • My Roles setting: "Set primary role" and "Set other role(s)" text links open compact modals for quick role editing; same links are duplicated in the My Champions tab header.
      • My Champions moved from Profile to the Champions tab with a persistent sub-navigation header (My Champions / Edit Champions toggle).
      • Champions tab now defaults to "My Champions" when navigated to.
      • Account settings editor: view email, game name, and tagline; edit first name and last name inline on the Profile page; and change password.
      • Admin Users shortcut moved from the left navigation bar into the Profile settings list (visible to admin users only); Admin Tools section removed from the nav.
      • Profile tab internal route renamed from "player-config" to "profile" for clarity.

      Coming Soon

      Deferred and planned follow-up work by workspace page.

      General

      • Champion core baseline data is now viewable in the admin workspace, but release ingestion is still manual and broader champion_core baseline adoption across champion/Riot lookup flows is still in progress (draftflow-129, draftflow-212).

      Tags

      • Re-evaluate migration scope for broader scoped rollout once the current tag-scope and tag-promotion UX has been validated in real use (draftflow-228).
      • Expand the scoped-promotion workflow beyond tags once the tag moderation UX is confirmed (draftflow-171).

      Compositions

      • Promotion moderation for scoped requirement/composition changes is still deferred (draftflow-171).

      Previous Release Notes

      Historical shipped changes from earlier versions.

      Version 0.7.6 (Shipped)

      • Player Slots section removed — all functionality (pick order badges, status badges, left/right reorder buttons, drag-to-reorder, exclusion and duplicate filtering) is now merged into Pool Snapshot cards.
      • Pool Snapshot cards now show a two-row header: summoner name + role dropdown on top, reorder arrows + pick order + status badge below.
      • Pool Snapshot champion lists now filter out excluded champions and champions already selected in other slots.
      • Advanced Controls restyled: compact 3-column layout with excluded champions list (click-to-toggle), active exclusion pills (click to remove), and scoring controls side by side.
      • Pool Snapshot filter lists, champion card grid, and excluded champions list now have consistent bordered styling for visual clarity.
      • Champion card grid now has a visible top border separator from the results count, preventing a cut-off appearance when scrolled.

      Version 0.7.5 (Shipped)

      • Champion tag editor redesigned: identity card and metadata now sit in a two-column row — the header (~25%) on the left, a bordered grouping container (~75%) on the right.
      • Roles and Damage Type are now displayed above Effectiveness in the metadata container, arranged side-by-side in a 50/50 centered grid.
      • "All roles same" checkbox renamed to "Apply tags to all selected Roles?" and moved into the Roles section header row.
      • Role profile selector relabeled from "Profile for:" to "UPDATE ROLE PROFILE:" and repositioned to the bottom of the metadata container.
      • Role profile selector pills are now equal width, sized to fit the longest role name (Jungle).
      • Saving champion tags now navigates back to the Champions grid automatically on success.
      • Damage type buttons in the tag editor now use the same colored pill design as champion cards instead of rectangular toggle buttons, and match the same size as role pills.
      • Fixed: champion card role pills now always render filled for single-role champions regardless of previously cached active role state.
      • Fixed: when multiple roles share the same profile, all role pills render filled; when profiles differ, only the leftmost pill is filled.

      Version 0.7.4 (Shipped)

      • Profile now supports per-champion familiarity grading with an explicit S/A/B/C scale in each role pool, including save/load persistence.
      • Composer tree generation now always expands to the end of the remaining draft instead of exposing a manual depth control.
      • Composer Advanced Controls now focuses tree tuning on `Min Candidate Score`, `Rank Goal`, and a global `redundancyPenalty` for max-count overflow instead of opaque candidate/composition weight grids.
      • Composer requirement visibility now shows clause-level match counts, missing coverage, redundancy overflow, and score contributions in Review/Inspect.
      • Composer top-branch cards now prioritize readable recommendation signals, with rank, viable-finish counts, primary benefits, and collapsed debug metrics instead of leading with raw engine stats.
      • Composer top-branch cards now surface total immediate coverage gain, explicit unmet clause coverage, and `+N more clause changes` indicators so near-tied recommendations are easier to tell apart.
      • Users workspace now summarizes scoped self/team/all access for Champion Tags, Champion Metadata, Requirements, and Compositions from the real authorization matrix, alongside the raw permission table.

      Version 0.7.3 (Shipped)

      • Champion cards now show role pills as clickable tab selectors; for champions with distinct per-role profiles, clicking a role switches the Damage Type and Effectiveness Focus display to that role's data.
      • Effectiveness Focus on champion cards always shows all three phases (Early, Mid, Late) including neutral.
      • Champion tag editor effectiveness profile redesigned: sliders replaced with Weak / Neutral / Strong pill buttons; damage and effectiveness now display as side-by-side columns; multi-role champions show a role tab row to edit each role's profile individually.
      • Login screen flash on page refresh eliminated for users with an active session.
      • Tab navigation flash on page refresh eliminated — the correct workspace is shown immediately on load without a brief flicker to Composer.
      • Champion results counter now shows "Results: N" instead of verbose sentence text.
      • Champions page card grid now has a visually separated container with gradient background and border to distinguish it from the filter section above.

      Version 0.7.2 (Shipped)

      • Users workspace now includes an admin-only Roles & Permissions matrix with a unified Roles section and a single Permission Assignments table that combines global and team-membership grants.
      • Team context is now active-team only; deprecated preferred/default-team persistence and related catalog wording were removed.
      • Roles labeling in the Users authorization matrix was clarified: global `member` now displays as `user`, and team membership roles display as `team member` and `team lead`.
      • Compositions now have explicit permission ids in the catalog and assignments (`compositions.read.global`, `compositions.write.global`) instead of being implied under requirements permissions.
      • Owner allowlisted accounts now resolve to admin access from email allowlist even when legacy stored DB role is `member`, and the Users page now includes an owner-only "Apply Admin to DB" action to persist the DB role back to `admin`.

      Version 0.7.1 (Shipped)

      • Composer can now generate trees from an empty setup; you no longer need a preselected champion to start expansion.
      • Empty-start generation now branches from the first role in your current pick order, creating root candidates from that role's available pool (subject to normal branch thinning).
      • Draft-context tree candidates are now role-exact per lane member: secondary-role pools are no longer mixed into future node options.

      Version 0.7.0 (Shipped)

      • Composer Setup now includes an Active Composition selector populated from created composition bundles.
      • Composer review and tree generation now evaluate selected composition requirements directly, including clause logic and per-clause constraints.
      • Legacy hardcoded conditional controls were removed from Composer, including advanced toggle conditionals and optional-check sections.

      Version 0.6.0 (Shipped)

      • Requirements and Compositions are now separate workspace pages so definition authoring and bundle management are split into focused flows.
      • Users was moved into a visually separated Admin Tools area near Profile/Logout to keep normal user navigation focused.
      • Requirement definitions now support CRUD for `name`, `definition`, and phase-2 style `rules` clauses (expr/minCount/maxCount/roleFilter).
      • Requirement rule entry now uses an in-form clause builder with side-by-side Tag, Damage Type, and Effectiveness Focus pickers, AND/OR joiners, min/max counts, role filters, and optional separate-champion constraints instead of requiring raw JSON input.
      • Requirement editor behavior now auto-closes after create/update so admins can immediately start a fresh requirement from the list view.
      • Requirement option rows now expose hover definitions/tooltips for tags and condition metadata.
      • Compositions now support CRUD for named bundles of requirement definitions, including active-composition selection.

      Version 0.5.2 (Shipped)

      • Navigation bar redesigned as a fixed full-height sidebar anchored to the left viewport edge with a smooth collapse/expand toggle; nav stays visible whether logged in or out.
      • Login, Register, Forgot Password, and Reset Password flows now appear as a centered card modal that overlays the nav layout instead of replacing the full page.
      • Register form now captures Game Name, Tagline, First Name, and Last Name alongside password; First Name and Last Name are persisted to the database. A Re-type Password field with live mismatch validation prevents mismatched submissions.
      • Password fields across all auth forms now include a show/hide toggle so users can verify what they have typed.
      • Change Password is available from the Profile page (Account section) and opens a dedicated modal with New Password and Re-enter New Password confirmation fields; a server-side change-password endpoint was added.
      • Updates and Report an issue moved to link-style items at the bottom of the nav; Report an issue now shows an external link indicator (↗).
      • Nav sidebar right-edge border radius removed so the nav sits flush against the left viewport edge.

      Version 0.5.1 (Shipped)

      • Champion metadata editing now supports an explicit "All selected roles are the same" option so one primary damage/effectiveness profile can be applied across every selected role.
      • The Tags workspace now stacks each tag row as name, then definition, then champion usage count for faster scanning.

      Version 0.5.0 (Shipped)

      • Tags now use a definition-first model: catalog entries are managed as `name + definition`, and category-based tag organization has been removed.
      • Champion editing was unified so tag assignment and champion role metadata live in one profile editor flow.
      • Champion role metadata now supports per-role primary damage type (`ad`, `ap`, `mixed`, `utility`) and per-role effectiveness (early/mid/late as weak/neutral/strong).
      • `utility` role profiles no longer satisfy damage-mix requirements in composition evaluation logic.

      Version 0.4.3 (Shipped)

      • The Champions filter panel was redesigned into a single collapsible section; the Tag Catalog was moved inside the filter grid as a peer field with an overlay panel so it doesn't expand the layout.
      • Active filter pills now appear inline next to the Filters toggle, showing a label per active filter with an instant-display list tooltip on hover and a visual highlight on mouse entry.
      • Filter fields each gained an inline × clear icon; a warning banner surfaces the mutual exclusivity of Include and Exclude tag filters.
      • The champion card grid is capped at three visible rows with vertical scrolling to keep the page compact.
      • Filter field text now uses consistent bold weight across all inputs, selects, and dropdowns.

      Version 0.4.2 (Shipped)

      • Player Slots and Pool Snapshot panels in the Composer were overhauled into a unified slot-based workflow; the separate Node Draft Order drag list was removed and replaced with inline ↑↓ reorder buttons on each slot row.
      • Pool Snapshot cards now show per-member, per-role champion lists with a text filter and scrollable view; clicking a champion directly sets it in the corresponding slot.
      • Slot rows detect and flag duplicate role assignments inline, and changing a slot's role resets the champion selection to prevent stale picks.
      • Draft context is now fetched on page load rather than only on team change, ensuring the Composer is always pre-populated without requiring user interaction.

      Version 0.4.1 (Shipped)

      • The Teams workspace now splits “Teams I’m On”, “Teams I manage”, and “Create new team” into separate tabs so roster management, invites, requests, and status summaries stay focused.
      • Team invites, join requests, and status feedback are front and center alongside a current-team summary pulled from your roster data.
      • Team name/tag uniqueness is hardened via the latest migration, ensuring no duplicate teams slip through.

      Version 0.4.0 (Shipped)

      • Team workspace exposes invitation controls (send/cancel, pending lists, and lead review) alongside existing join-request moderation.
      • Team creation and edits enforce case-insensitive uniqueness for names and tags to prevent duplicates.
      • The Champion Tag editor now understands team scope so team leads can manage team-specific composition metadata.

      Version 0.3.1 (March 2, 2026)

      • Aligned the champion tag editor "Human reviewed" checkbox control for consistent label/checkbox layout
      • Aligned the Composition Requirements "Set as active profile" checkbox control with the Requirements editor layout
      • Reorganized this Updates page so shipped changes and roadmap items are split into separate sections

      Version 0.3.0

      • Owner admin can now manage team roster actions (including member removal) even when not assigned as a team lead
      • Users workspace now supports one-time admin Riot ID corrections for mistyped Game Name/Tagline entries
      • Added admin-only Users workspace to view all accounts and manage `member` / `global` permissions (draftflow-153)
      • Added Composition Requirements workspace with CRUD profiles and active-profile selection for Composer defaults (draftflow-155, draftflow-156)
      • Added champion composition-tag review flag support in the editor and champion cards (draftflow-154)
      • Global editor role now supports global champion-tag and tag-catalog writes without full admin access (draftflow-153)
      • Users workspace now supports permanent deletion of non-owner accounts for demo-user cleanup
