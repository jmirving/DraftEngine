      Updates & Roadmap

      Latest shipped changes plus known gaps and planned follow-up grouped by workspace page.

      What's New

      Shipped updates that are available now.

      Version 0.4.1 (Shipped)

      • The Teams workspace now splits “Teams I’m On”, “Teams I manage”, and “Create new team” into separate tabs so roster management, invites, requests, and status summaries stay focused.
      • Team invites, join requests, and status feedback are front and center alongside a current-team summary pulled from your roster data.
      • Team name/tag uniqueness is hardened via the latest migration, ensuring no duplicate teams slip through.

      Version 0.4.0 (Shipped)

      • Team workspace exposes invitation controls (send/cancel, pending lists, and lead review) alongside existing join-request moderation.
      • Team creation and edits enforce case-insensitive uniqueness for names and tags to prevent duplicates.
      • The Champion Tag editor now understands team scope so team leads can manage team-specific composition metadata.

      Version 0.3.1 (March 2, 2026)

      • Aligned the champion tag editor "Human reviewed" checkbox control for consistent label/checkbox layout
      • Aligned the Composition Requirements "Set as active profile" checkbox control with the Requirements editor layout
      • Reorganized this Updates page so shipped changes and roadmap items are split into separate sections

      Version 0.3.0

      • Owner admin can now manage team roster actions (including member removal) even when not assigned as a team lead
      • Users workspace now supports one-time admin Riot ID corrections for mistyped Game Name/Tagline entries
      • Added admin-only Users workspace to view all accounts and manage `member` / `global` permissions (draftflow-153)
      • Added Composition Requirements workspace with CRUD profiles and active-profile selection for Composer defaults (draftflow-155, draftflow-156)
      • Added champion composition-tag review flag support in the editor and champion cards (draftflow-154)
      • Global editor role now supports global champion-tag and tag-catalog writes without full admin access (draftflow-153)
      • Users workspace now supports permanent deletion of non-owner accounts for demo-user cleanup

      Coming Soon

      Deferred and planned follow-up work by workspace page.

      Version 0.4.2 (Upcoming)

      • Team-scoped composition requirement workflows plus promotion moderation for leads are still in flight (draftflow-171).
      • Champion-tag promotion review controls and governance surfaces remain pending (draftflow-170).
      • Invite/request lifecycle polish, including improved lead-review telemetry, is on deck (draftflow-169).

      General

      • Champion catalog refresh checks are still manual; automatic release ingestion is not enabled yet (draftflow-129)
      • Admin safeguards and governance guardrails still need hardening for high-impact global edits (draftflow-147)

      Profile

      • Set your champion familiarity level in your personal pool (draftflow-144)
      • Get champion pool suggestions based on your real League match history (draftflow-145)
      • Live Riot champion stats in Profile remain a placeholder until the full implementation is shipped (draftflow-146)

      Composer

      • Composer currently starts from the active Composition Requirement profile, but per-team and per-user profile overlays are still deferred (draftflow-160)

      Teams

      • Teams member actions require exact Riot ID entry (`GameName#Tagline`); search/autocomplete is not implemented yet (draftflow-139)
      • Join request controls currently require manual refresh actions for discover/review lists; live auto-refresh is deferred.
      • Promotion moderation dashboards for team-scoped champion tags and requirement toggles remain in progress (draftflow-170, draftflow-171).

      Champions

      • Champion review state currently tracks boolean + reviewer metadata only; audit history and review notes are deferred (draftflow-159)
      • Ship champion auto-ingest now that global champion-tag persistence is in place (draftflow-129)

      Tags

      • Global editor permissions currently target tag-catalog and champion-tag writes only; finer-grained policy controls are deferred (draftflow-161)

      Users

      • User search, filtering, and bulk permission updates are not implemented yet (draftflow-157)
      • User deletion is currently hard-delete only; restore/undo and deletion audit history are deferred

      Composition Requirements

      • Requirement profiles support toggle-based rules today; custom rule definitions and version history are deferred (draftflow-158)
